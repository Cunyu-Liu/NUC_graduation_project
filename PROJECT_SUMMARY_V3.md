### 2. 博士级提示词工程 (`src/prompts_doctoral.py`)
**代码量**: 800+行
**提升**: 300%深度提升

#### 摘要生成提示词 - 7层结构
1. 研究背景与动机(2-3句)
2. 现有方法的局限性(1-2句)
3. 核心方法与创新(3-4句)
4. 技术细节(1-2句)
5. 实验设计与数据集(1-2句)
6. 主要结果与性能(2-3句)
7. 结论与影响(1-2句)

**质量标准**:
- 学术严谨性: 规范语言,专业术语
- 逻辑连贯性: 层层递进,形成闭环
- 信息密度: 每句话有实质内容
- 深度与洞察: 揭示研究本质
- 长度控制: 中文500-700字,英文350-500词

#### 要点提取提示词 - 12个深度类别

| 类别 | 数量 | 深度要求 | 状态 |
|-----|------|---------|------|
| 核心创新点 | 4-6个 | 包含技术细节和量化指标 | ✅ |
| 研究空白与动机 | 2-4个 | 指出具体空白,分析原因 | 🆕 |
| 理论框架与假设 | 3-5个 | 提取理论基础,说明适用边界 | 🆕 |
| 主要方法与技术 | 5-8个 | 原理说明,关键设计 | ✅ |
| 实验设计与数据集 | 4-6个 | 实验类型,数据集信息 | ✅ |
| 数据集详细分析 | 2-4个 | 规模,来源,标注,质量 | 🆕 |
| 主要结论与性能 | 4-6个 | 量化结果,统计显著性 | ✅ |
| 统计分析与显著性 | 2-3个 | 统计方法,p值,置信区间 | 🆕 |
| 与相关工作的对比 | 3-5个 | 本质区别,借鉴改进 | ⚡ |
| 可复现性与实现细节 | 3-4个 | 代码可用性,实现细节 | 🆕 |
| 学术贡献与影响 | 3-5个 | 理论贡献,技术推动 | ✅ |
| 研究局限性与未来方向 | 3-5个 | 诚实具体,改进方向 | ⚡ |

#### 主题分析提示词 - 8个维度 🆕
1. 研究领域定位(一级/二级学科/研究方向/交叉学科)
2. 核心主题词(5-8个,降序排列)
3. 研究范式与方法论
4. 问题类型与求解策略
5. 技术栈与工具
6. 应用场景与领域
7. 理论基础与依赖
8. 研究前沿性与成熟度

#### 研究空白挖掘提示词 🆕
- 5类空白: 方法论、理论、数据、应用、评估
- 每个空白包含: 描述、重要性、难度、潜在方法、预期影响

#### 趋势预测提示词 - 7个维度 ⚡
1. 当前研究热点分析
2. 短期趋势预测(1-2年)
3. 中长期趋势预测(3-5年)
4. 跨领域机会识别
5. 研究空白与机会
6. 资源配置建议
7. 风险与挑战

## 🚀 使用方式

### Python API
```python
from src.doctoral_analyzer import DoctoralAnalyzer

analyzer = DoctoralAnalyzer(model="glm-4-plus")

# 单篇分析
result = analyzer.analyze_single_paper("paper.pdf")

# 多篇分析
result = analyzer.analyze_multiple_papers(
    ["paper1.pdf", "paper2.pdf", "paper3.pdf"],
    enable_clustering=True,
    enable_gap_mining=True
)

# 对比分析
comparison = analyzer.compare_papers(
    ["paper1.pdf", "paper2.pdf"]
)
```

### CLI
```bash
python -m src.doctoral_analyzer analyze paper.pdf
```

### Web界面
```bash
# 后端
python app.py

# 前端
cd frontend && npm run serve
```
